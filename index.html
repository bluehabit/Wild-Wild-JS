Bracket Notation

Bracket notation is a way to get a character at a specific index within a string.

	
		var myName = 'Chris';
		myName[0] //-> 'C';
		myName[myName.length - 1] //-> 's'
		//getting the last character of a string

		var lastChar = myName[myName.length - 1];
		//can even save the last character to a variable if required

Strings are Immutable

In Javascript, String values are immutable. This means they cannot be changed onced created. For example: 

	
		var myName = 'Bill';
		myName[0] = 'J';
		console.log(myName); //-> 'Bill'

Cannot change the value of myStr to 'Jill', because the contents of myStr cannot be altered. The only way to change myStr would be to assign it with a new string. But you cannot change the individual characters of a string literal. 

	
		var myName = 'Chris';
		//reassigning the string literal 'Chris' to myName
The only mutable types in JS are Objects and Arrays.

String.substring()

The substr(start,length) function is similar to the slice function, only the 2nd parameter represents the length of the sub-string. If it is not provided then the rest of the string is returned. Although both methods do function similarly. 

		
			var myString = 'hello world banana';
			myString.substr(0,4);
			//hell

			var myString = 'hello world banana';
			myString.substr(6);
			//world banana

replace(str1, str2)

The replace function replaces the first occurence of str1 in a string with str2. If you want to replace every occurence, it would be wise to use a regular expression.

For example, /abc/g replaces every occurrence of abc in the original string.

		
			var myString = 'a faintly glimmering candle dances in the window';
			myString.replace('glimmering', 'flickering');
			myString //-> "a faintly flickering candle dances in the window""

Special Note when you use replace it assigns a new string literal to the variable.

Number.EPSILON

Special Note Numbers that are not integers are often referred to as floating point numbers. Operations on these numbers may not always result in the exact value because computers have difficulties representing them. For example, 0.6 + 0.3 evaluates to 0.8999999999999999 instead of 0.9. One way to fix such logic is to calculate the difference between the actual result and the expected result and determine whether the difference is very small, i.e., close to zero. In JavaScript, Number.EPSILON represents a number that is very close to zero. We can check if the difference is smaller than Number.EPSILON to verify that it is small enough. 

		
			var x = 0.6;
			var y = 0.3;
			var diff = (x + y) - 0.9; //the difference between (x+y) and 0.9
			var absDiff = Math.abs(diff); // absolute difference
			//the absolute difference is very small, meaning (x+y) and 0.9 are almost equal
			
			if (absDiff < Number.EPSILON){ 
  				console.log("Inside if");
			}

In this example, the variable diff holds the difference between x + y and 0.9. Then we calculate the absolute difference using the Math.abs(...) function. If this absolute difference is smaller than Number.EPSILON, we can say that x + y and 0.9 are almost equal.

Math.random

Variant 1: 
		
			var number = Math.floor(Math.random() * 6);
			console.log(number)

Can produce 0 which would be ideal for something such as a RGB guessing game where you want the number 0 to be a possibility

Variant 2: 
		
			var number = Math.floor(Math.random() * 6) + 1;
			console.log(number)

Generates a random # between 1 and 6. Unable to produce 0. Ideal for something like a dice roll.
