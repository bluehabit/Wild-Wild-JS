Number.EPSILON

Special Note Numbers that are not integers are often referred to as floating point numbers. Operations on these numbers may not always result in the exact value because computers have difficulties representing them. For example, 0.6 + 0.3 evaluates to 0.8999999999999999 instead of 0.9. One way to fix such logic is to calculate the difference between the actual result and the expected result and determine whether the difference is very small, i.e., close to zero. In JavaScript, Number.EPSILON represents a number that is very close to zero. We can check if the difference is smaller than Number.EPSILON to verify that it is small enough. 

		
			var x = 0.6;
			var y = 0.3;
			var diff = (x + y) - 0.9; //the difference between (x+y) and 0.9
			var absDiff = Math.abs(diff); // absolute difference
			//the absolute difference is very small, meaning (x+y) and 0.9 are almost equal
			
			if (absDiff < Number.EPSILON){ 
  				console.log("Inside if");
			}

In this example, the variable diff holds the difference between x + y and 0.9. Then we calculate the absolute difference using the Math.abs(...) function. If this absolute difference is smaller than Number.EPSILON, we can say that x + y and 0.9 are almost equal.

Switch Statements

Example 1: 
		
function caseInSwitch(val){
	var answer = '';
		switch(val){
			case 1:
				answer = 'alpha';
				break;
			case 2:
				answer = 'beta';
				break;
			case 3:
				answer = 'gamma';
				break;	
			case 4:
				answer = 'delta';
				break;	
			}
			return answer;
		}

		caseInSwitch(1);
		//alpha

Special Note It is best practice to only use break inside of switch statements. If you have many options, its best practice to use a switch statement. Using break outside of switch statements introduces unconditional jumps (also known as go to's) which leads to spaghetti code.

Default Option

In a switch statement you may not be able to specify all possible values as case statements. Instead, you can add the default statement which will be executed if there is no matching case statement found.

Example 2: 
		
function switchOfStuff(val){
	var answer = '';
		switch(val){
			case 'a':
				answer = 'apple';
				break;
			case 'b':
				answer = 'bird';
				break;
			case 'c':
				answer = 'cat';
				break;	
			default:
				answer = 'stuff';
			}
			return answer;
		}

Functions let us wrap bits of code into REUSABLE packages. They are on of the building blocks of JS.

Then call it

Functions are designed to RETURN something. The return keyword is a way to CAPTURE THE VALUE coming out of the function so you can do something with it later. Think of each function as its own factory. Once the function is called and executed it returns a pacakge.

Special Note: Can only ever return one thing from a function; however, you can use conditional statements such as if to route what is returned.
